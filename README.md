# Calculus of Learning - Experimental Codebase

Welcome to the official experimental codebase for the paper **"Learning as Differentiation, Imagining as Integration"**. This repository is designed to empirically verify the mathematical hypotheses presented in the paper across three core experiments:
- **E1:** Rollout drift and error laws of imagination (Integrator scaling).
- **E2:** The relationship between representational frame freedom and out-of-distribution (OOD) generalization.
- **E3:** Zero-shot latent plan transfer across coordinate frames.

Below is a step-by-step guide to setting up the environment and reproducing the results, designed to be accessible even if you are new to the codebase.

---

## 🛠 1. Installation and Setup

This project uses [`uv`](https://docs.astral.sh/uv/) for lightning-fast dependency management and `JAX` as the primary deep learning framework.

### Step 1: Install `uv`
If you haven't installed `uv` yet, run the following command:
```bash
# MacOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### Step 2: Create Environment and Install Dependencies
Navigate to the root of the project and run the following command. This will automatically create a virtual environment (`.venv`) and install all packages specified in `pyproject.toml`:
```bash
uv sync
```

### Step 3: Install JAX (GPU / CPU)
By default, `uv sync` installs the CPU version of JAX. If you have an NVIDIA GPU and wish to leverage hardware acceleration, install the CUDA version of JAX using:
```bash
uv pip install -U "jax[cuda12]"
```
*(If you are running on a CPU-only machine or a Mac, you can skip this step).*

### Step 4: Weights & Biases Setup (Optional)
If you wish to log and track your experiments using `wandb`, copy the example environment file and add your authentication token:
```bash
cp .env.example .env
```

## 🚀 2. Running the Experiments

**Note:** All commands below should be executed from the project root. We use `uv run python` to automatically use the created virtual environment. On Linux/macOS, remember to prefix your commands with `PYTHONPATH=$(pwd)` to avoid import errors.

### 🧪 Experiment E1: Error Laws (Toy Environment)
This experiment simulates a linear system with a Lipschitz constant $L=1.05$. It trains three models (1-step M1, 4-step M2, and 8-step M3), carefully matches their 1-step errors ($\epsilon$), and plots the compounding rollout error to empirically verify the Integrator Law.

**Step 1: Train the predictors**
```bash
PYTHONPATH=$(pwd) uv run python experiments/e1_error_laws/train_toy.py --l_value 1.05
```
*You will see the training progress as the models match the `Val 1-step MSE`. Checkpoints will be saved in `outputs/checkpoints/e1/`.*

**Step 2: Evaluate and Plot**
```bash
PYTHONPATH=$(pwd) uv run python experiments/e1_error_laws/eval_toy.py --l_value 1.05
```
*This script fits the accumulated error data $E(h)$ against three curves (Integrator, Linear, Power). In the final logs, you should see `Integrator Law won the AIC test` with $L_{fit} \approx 1.05$.*  
👉 **Result:** The resulting plot is saved at `outputs/plots/e1_error_laws_L1.05.png`.

---

### 🧪 Experiment E2: Frame Freedom vs OOD Generalization
This experiment investigates how the degree of residual frame freedom (controlled by different joint-embedding regularizers) impacts out-of-distribution dynamics generalization.

**Run the full sweep:**
We provide a bash script to automate the training sweep across environments:
```bash
bash scripts/run_e2_frame_freedom.sh
```

---

### 🧪 Experiment E3: Plan Transfer Across Frames
This experiment demonstrates that a latent plan generated by Model A can be transferred zero-shot to Model B using only a simple mathematical transformation (Linear Map), without retraining Model B.

**Test different mapping strategies:**
Run the provided bash script and specify the mapping type:
```bash
# Naive baseline (direct copy, expected to fail)
bash scripts/run_e3_plan_transfer.sh --env cartpole-swingup --mapping identity

# Orthogonal Procrustes rotation (Predicted by LeJEPA theory to be sufficient)
bash scripts/run_e3_plan_transfer.sh --env cartpole-swingup --mapping orthogonal

# Standard Affine transformation
bash scripts/run_e3_plan_transfer.sh --env cartpole-swingup --mapping affine

# Semantic Alignment via Canonical Correlation Analysis (CCA)
bash scripts/run_e3_plan_transfer.sh --env cartpole-swingup --mapping cca
```

│   └── visualization/        # Plotting and statistical metric calculation (AIC, R^2)

│   ├── e2_frame_freedom/
